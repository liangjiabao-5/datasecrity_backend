# 现场测评与汇总分析功能修改说明

更新时间：2026-06-02

## 1. 现场测评页面

### 1.1 删除内容

现场测评表格、编辑弹窗、保存接口载荷、导出模板和导出文件中删除以下字段：

- `problemDescription`：问题描述
- `remediationSuggestion`：整改建议
- `riskTypes`：风险类型

后端导入模板和导出模板也需要同步删除以上三列，导入解析时不再读取这三列。

### 1.2 字段改名和位置调整

- 原 `evaluationResult` 字段页面展示名称由“评估结果”改为“符合情况”。
- 原 `evaluationRecord` 字段页面展示名称由“评估记录”改为“评估结果”。
- 表格和编辑弹窗中，“评估结果”（`evaluationRecord`）展示在“符合情况”（`evaluationResult`）之前。

### 1.3 接口影响

保存现场测评记录接口：

```http
PUT /api/v1/projects/{projectId}/evaluation/items/{itemId}/record
```

调整后的请求体只保留：

```json
{
  "evaluationResult": "NON_COMPLIANT",
  "evaluationRecord": "未见明确数据安全总体策略"
}
```

查询、导入、导出现场测评数据时，建议后端仍返回 `evaluationResult` 和 `evaluationRecord` 原字段名，但页面展示按新名称处理。

## 2. 汇总分析页面

### 2.1 数据来源调整

“数据安全风险源清单”来源于现场测评表中 `evaluationResult = NON_COMPLIANT` 或 `evaluationResult = PARTIAL` 的数据。

一条“不符合”或“基本符合”的现场测评记录对应一条风险记录，并由同一条风险记录投影展示到：

- 数据安全风险源清单
- 数据安全风险清单
- 数据安全风险处置建议

### 2.2 数据安全风险源清单

新增列：

- 检查要点：对应现场测评 `checkPoint`
- 评估结果：对应现场测评 `evaluationRecord`

修改列名：

- “涉及数据”改为“涉及的数据及类型、级别”
- “处理活动”改为“涉及的数据处理活动”

删除列：

- 标识

### 2.3 数据安全风险清单

新增列：

- 在“发生可能性”后增加“风险源描述”

修改列名和展示内容：

- “涉及数据”改为“涉及的数据及类型、级别”
- “调整原因”改为“涉及的数据处理活动”，展示风险记录中的 `relatedActivities`

删除列：

- 标识

### 2.4 数据安全风险处置建议

新增/调整列：

- “风险等级”展示在“风险源描述”之后

删除列：

- 标识

## 3. 后端存储建议

由于三个汇总分析标签页是一对一关系，建议后端将原风险源、风险清单、处置建议三类数据合并为一张风险汇总记录表，例如 `project_risk_summary_records`。

建议核心字段：

```text
id
project_id
evaluation_item_id
source_item_code
check_point
evaluation_result
evaluation_record
risk_types
risk_description
risk_source_description
related_data
related_activities
harm_level
possibility_level
risk_level
remediation_suggestion
manual_adjusted
created_at
updated_at
```

兼容策略：

- 现有三个查询接口可以继续保留，但都从同一张风险汇总记录表读取并按页面需要投影返回，支持 `pageNo`、`pageSize` 并统一返回 `{ list, pageNo, pageSize, total }`。
- 现有三个修改接口可以继续保留，但都更新同一条风险汇总记录。
- `riskSourceId`、`riskItemId`、`suggestionId` 可统一映射为同一个 `riskRecordId`。

## 4. 刷新统计数据逻辑

刷新汇总分析时：

- 从现场测评表筛选 `evaluationResult = NON_COMPLIANT` 或 `evaluationResult = PARTIAL` 的记录。
- 每条不符合或基本符合记录生成或更新一条风险汇总记录。
- 根据 `sheetName`、评估类别、评估子类、评估项匹配国标风险源模板，并填充评估类别、评估子类、风险描述、整改建议、风险源描述、风险源类型和风险类型。
- 按现场测评“分数计算”同一口径计算当前项目得分，并根据项目绑定评分模型的 `possibilityRanges` 自动填充 `possibilityLevel`。
- 发生可能性区间按 `min <= score < max` 判断；为覆盖满分，`score = 100` 且区间 `max = 100` 时命中该区间。
- 已人工修改的风险字段建议保留，不被自动刷新覆盖。
- 如果现场测评记录从“不符合/基本符合”改为其他符合情况，对应风险记录建议删除、失效或标记为非当前风险。

## 5. 前端已同步的变化

- 现场测评页面已删除问题描述、整改建议、风险类型列和编辑项。
- 现场测评保存请求已不再提交 `problemDescription`、`remediationSuggestion`、`riskTypes`。
- 汇总分析三张表已按新列名和新列顺序展示。
- 前端 mock 数据已改为单张风险汇总记录存储，并从“不符合”的现场测评记录生成一对一风险数据。
