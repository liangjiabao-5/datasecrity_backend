# 测试验收方案

## 1. 测试目标

测试目标是验证系统能按照需求说明和原型完成数据安全风险评估报告编制全流程，并保证核心计算、导入导出、风险清单、报告生成和权限审计可靠。

## 2. 测试范围

| 模块 | 测试内容 |
| --- | --- |
| 项目管理 | 创建、编辑、查看、删除、开始评估、状态筛选、项目模板 |
| 项目基本信息 | 项目信息、法规标准、被评估单位、联系人、导入导出 |
| 评估方案 | 评估团队、被评估方团队、关注点、差距项 |
| 信息调研 | 业务系统、数据资产、个人信息、重要/核心数据、处理活动、防护措施 |
| 现场测评 | 检查项、评估结果、评估记录、批量修改、分数计算、案例建议 |
| 汇总分析 | 风险源、风险清单、处置建议、刷新统计数据、人工修改 |
| 报告管理 | 报告生成、状态轮询、下载、删除、分页 |
| 知识库管理 | 模板、模型、矩阵、建议、案例、版本、停用 |
| 大模型辅助 | 案例匹配、建议生成、脱敏、人工确认、降级 |

## 3. 测试类型

- 单元测试：计算规则、区间校验、矩阵计算、状态流转。
- 接口测试：所有核心接口的成功、失败、权限、异常。
- 前端功能测试：页面交互、标签页、按钮、表单校验、弹窗。
- 集成测试：项目全流程闭环。
- 文件测试：上传、下载、导入、导出、报告生成。
- 权限测试：不同角色访问、按钮显示、接口拒绝。
- 回归测试：需求变更后核心流程复测。

## 5. 非功能验收

| 项目 | 验收标准 |
| --- | --- |
| 列表性能 | 常规列表 1 秒内返回 |
| 报告生成 | 采用异步任务，页面不阻塞 |
| 文件安全 | 未授权用户不能下载项目文件 |
| 权限 | 无权限用户不能看到或调用新增、编辑、删除、生成接口 |
| 审计 | 关键操作均有审计记录 |
| 稳定性 | 大模型不可用、报告生成失败、导入失败不影响其他模块 |

- 测试用例清单。
- 接口测试报告。
- 前端功能测试报告。
- 报告生成样例。

- 权限测试结果。
- 缺陷清单和修复记录。

## 7. 验收通过条件

- 主流程端到端跑通。
- P0/P1 缺陷为 0。
- 报告生成内容与项目最终确认数据一致。
- 评分计算和风险等级计算通过业务复核。
- 权限和文件下载不存在越权问题。
- 用户认可页面操作和报告输出。
