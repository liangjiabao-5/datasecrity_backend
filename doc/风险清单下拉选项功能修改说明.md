# 风险清单下拉选项功能修改说明

更新时间：2026-07-16

## 一、调整范围

本次只调整“数据安全风险源清单”的以下两列为下拉勾选：

| 页面区域 | 字段 | 后端字段 | 前端处理 |
| --- | --- | --- | --- |
| 数据安全风险源清单 | 涉及的数据及类型、级别 | `relatedData` | 下拉多选，可编辑保存 |
| 数据安全风险源清单 | 涉及的处理活动 | `relatedActivities` | 下拉多选，可编辑保存 |
| 数据安全风险清单 | 涉及的数据及类型、级别 | `relatedData` | 字符串展示，不可编辑 |
| 数据安全风险清单 | 涉及的处理活动 | `relatedActivities` | 字符串或标签展示，不可编辑 |

## 二、下拉选项接口

```http
GET /api/v1/projects/{projectId}/risk-form-options
```

该接口供“数据安全风险源清单”编辑下拉框使用。

响应示例：

```json
{
  "code": "SUCCESS",
  "message": "Operation succeeded.",
  "data": {
    "relatedDataOptions": [
      "个人信息-联系方式/L2",
      "交易数据/L3"
    ],
    "relatedActivityOptions": [
      "收集",
      "传输",
      "存储",
      "使用和加工",
      "提供",
      "公开",
      "删除"
    ]
  },
  "traceId": "..."
}
```

`relatedDataOptions` 生成规则：

1. 来源为本项目“信息调研 - 数据资产调研”中“数据资产情况”列表。
2. 后端按每行 `dataCategory` 和 `dataLevel` 拼接为 `数据类别/数据级别`。
3. 拼接前会去除首尾空白，重复结果只返回一次。
4. `dataCategory` 或 `dataLevel` 为空的行不进入下拉选项。

## 三、风险源清单保存方式

风险源清单保存接口保持不变：

```http
PUT /api/v1/projects/{projectId}/risk-sources/{riskSourceId}
```

推荐提交格式：

```json
{
  "relatedData": ["个人信息-联系方式/L2", "交易数据/L3"],
  "relatedActivities": ["收集", "存储"]
}
```

说明：

- `relatedData` 前端可提交字符串或数组。提交数组时，后端会按 `、` 拼接后保存，接口返回仍为字符串。
- `relatedActivities` 前端提交数组，后端按数组保存，接口返回数组。
- 编辑回显时，如果多选组件需要数组值，前端可把 `relatedData` 按 `、` 拆分后绑定到组件。
- 保存成功后以前端使用接口返回值刷新当前行，不要继续展示本地临时值。

## 四、风险清单展示方式

数据安全风险清单接口保持只读展示这两列：

```http
GET /api/v1/projects/{projectId}/risk-items?pageNo=1&pageSize=10
PUT /api/v1/projects/{projectId}/risk-items/{riskItemId}
```

前端要求：

1. `risk-items` 列表中的 `relatedData` 直接作为字符串展示。
2. `relatedActivities` 可按数组渲染为文本或标签，但不要提供编辑控件。
3. 保存风险清单时不要提交 `relatedData`、`relatedActivities`。
4. 即使前端误传这两个字段，后端也不会保存，接口返回仍以数据库原值为准。

## 五、前端交互建议

1. 进入汇总分析页或切换到风险源清单页签时，调用一次 `risk-form-options`。
2. 风险源清单的两个字段使用支持勾选的多选下拉框。
3. `relatedDataOptions` 为空时，提示评估人先到“信息调研 - 数据资产调研”维护数据类别和数据级别。
4. 如果评估人在信息调研页新增或修改数据资产，回到汇总分析页时需要重新拉取下拉选项。
5. Excel 导入导出格式本次不需要调整。

## 六、验收点

1. 数据资产中相同“数据类别/数据级别”组合只在下拉中出现一次。
2. 风险源清单能通过下拉勾选保存 `relatedData`、`relatedActivities`。
3. 风险清单两列不出现编辑控件，保存时不提交这两个字段。
4. 处理活动下拉项顺序为：收集、传输、存储、使用和加工、提供、公开、删除。
