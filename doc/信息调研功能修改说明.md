# 信息调研功能修改说明

更新时间：2026-05-28

## 修改目标

前端信息调研页面已按 `原型/信息调研.html` 调整为 4 个标签页：

- 业务和信息系统调研
- 数据资产调研
- 数据处理活动调研
- 安全防护措施调研

## 删除的前端功能

1. 删除独立标签页：
   - 个人信息
   - 重要数据
   - 核心数据

2. 删除数据处理活动的旧版列表式维护界面：
   - 旧界面为 `activityType / scenario / method / scale / involvesPersonalInfo / protectionMeasures / description` 的行列表。
   - 新界面改为原型中的分段问卷，不再以单行活动记录作为主要录入方式。

3. 删除业务和信息系统的旧版列表+弹窗维护方式：
   - 新界面改为单个调研表单，字段直接在页面内编辑。

## 修改的前端功能

### 1. 业务和信息系统调研

页面内容调整为原型表单字段：

- 信息系统名称
- 业务功能描述
- 业务服务对象
- 业务用户规模
- 业务覆盖地域
- 网络安全等级保护备案等级
- 业务相关部门信息
- 上传拓扑图
- 上传数据流转图

当前前端仍兼容旧接口 `GET/POST/PUT /projects/{projectId}/survey/business-systems`，取列表第一条作为当前项目的业务系统调研表单。

### 2. 数据资产调研

页面内包含 4 张表：

- 数据资产情况
- 个人信息情况
- 重要数据情况
- 核心数据情况

原独立的个人信息、重要数据、核心数据标签已合并到数据资产调研标签内。

建议后端继续保留或聚合以下数据能力：

- `data-assets`
- `personal-info`
- `important-data`
- `core-data`

其中 `personal-info / important-data / core-data` 建议补齐字段：

- `dataName`
- `dataCategory`
- `scale`
- `dataSource`
- `businessFlow`
- `sensitivity`（个人信息需要）

### 3. 数据处理活动调研

页面改为原型问卷结构，包含：

- 涉及处理活动类型
- 数据收集情况
- 数据传输情况
- 数据存储情况
- 数据使用和加工情况
- 数据提供情况
- 数据公开情况
- 数据删除情况

建议后端新增整体保存接口：

```http
GET /api/v1/projects/{projectId}/survey/processing-activity-survey
PUT /api/v1/projects/{projectId}/survey/processing-activity-survey
```

前端已临时兼容旧 `processing-activities` 列表接口；后端完成新接口后，前端会优先使用新接口。

### 4. 安全防护措施调研

页面从旧版 13 个文本框改为原型中的 22 个问卷项，主要包括：

- 等级保护测评
- 商用密码应用安全性评估
- 近 3 年安全检测/风险评估
- 历史问题整改
- 专职数据安全管理组织
- 数据安全负责人
- 安全设备
- 安全策略更新频率
- 身份鉴别措施
- 最小权限原则
- 漏洞扫描频率
- 高危漏洞修复周期
- 远程管理工具
- 远程访问权限分配对象
- 系统口令复杂度要求
- 传输加密情况
- 存储加密情况
- 脱敏、去标识化情况
- 近 1 年检测到的威胁类型
- 数据安全事件发生频率
- 数据安全事件类型
- 事件详情描述

建议后端将 `security-protection` 接口的数据结构调整为可保存结构化问卷字段，支持字符串、单选值和多选数组。

## 后端调整建议

1. 信息调研页面导航只保留 4 个标签，不再按前端菜单暴露个人信息、重要数据、核心数据为独立页面。
2. 数据资产相关专项数据可以继续复用旧接口，但应按新字段补齐返回和保存。
3. 数据处理活动建议新增整体问卷接口，旧 `processing-activities` 行记录接口可保留为兼容接口或后续下线。
4. 安全防护措施接口建议支持原型 22 项结构化字段，避免将多选内容压缩为单个文本字段。

## 后端处理结果

- 未删除现有信息调研表和旧接口，保留旧接口用于前端兼容和历史数据读取。
- 已补齐 `personal-info`、`important-data`、`core-data` 新字段：`dataName`、`dataCategory`、`scale`、`dataSource`、`businessFlow`，其中个人信息额外支持 `sensitivity`。
- 已新增整体问卷接口：
  - `GET /api/v1/projects/{projectId}/survey/processing-activity-survey`
  - `PUT /api/v1/projects/{projectId}/survey/processing-activity-survey`
- 已新增 `processing_activity_survey` 表，用于保存数据处理活动分段问卷。
- `security-protection` 继续使用结构化 JSON 问卷保存，支持字符串、单选值和多选数组。
