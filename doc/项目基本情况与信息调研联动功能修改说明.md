# 项目基本情况与信息调研联动功能修改说明

更新时间：2026-06-08

## 修改范围

本次变更涉及以下前端页面：

- 项目基本情况页面 - 项目基本情况标签页
- 信息调研页面 - 业务和信息系统调研标签页
- 信息调研页面 - 数据处理活动调研标签页
- 信息调研页面 - 安全防护措施调研标签页
- 创建项目页面、项目管理页面

## 删除或禁用的前端能力

1. 删除“创建项目成功后自动进入项目基本信息页面”的跳转行为。
   - 创建项目成功后统一跳转到项目管理页面。
   - 只有在项目管理页面点击“开始评估”后，才进入 `/projects/{projectId}/basic`。

2. 禁用未勾选数据处理活动类型对应环节的填写能力。
   - 未勾选“收集、传输、存储、使用和加工、提供、公开、删除”中的某一项时，对应环节表单置灰且不可编辑。
   - 保存前前端会清空未勾选环节的字段，避免后端保存未涉及活动的残留内容。

3. 禁用安全防护措施中“否”状态下补充输入框的填写能力。
   - 对“是/否”单选问题，只有选择“是”时对应补充输入框可编辑。
   - 选择“否”时补充输入框置灰不可编辑，保存前前端会清空该补充字段。

## 修改的前端能力

1. 项目基本情况页日期选择器中文化。
   - Element Plus 全局 locale 已改为中文。
   - 评估开始日期、评估结束日期的月份和周几显示中文。

2. 业务和信息系统调研上传文件类型扩展。
   - 上传拓扑图支持图片文件和 `.vsdx` 文件。
   - 上传数据流转图支持图片文件和 `.vsdx` 文件。
   - 前端 accept 和扩展名校验已包含 `.vsdx`。

3. 数据处理活动调研勾选项与七个填写环节联动。
   - `involvedActivities` 继续作为涉及处理活动类型字段。
   - 七个枚举值仍为：`COLLECT`、`TRANSFER`、`STORE`、`USE_PROCESS`、`PROVIDE`、`PUBLIC`、`DELETE`。
   - 前端将 `involvedActivities` 作为权威开关，只提交已勾选环节的有效内容。

4. 安全防护措施调研补充输入框位置和启用条件调整。
   - 有补充输入框的“是/否”单选项，输入框显示在“是”选项后面。
   - 控制值不是“是”时，补充字段为空字符串。
   - “事件详情描述”仍按自身选项控制，选择“事件描述”时可填写事件描述。

## 后端配合建议

1. 上传接口需要同步放行 `.vsdx`。
   - `POST /api/v1/projects/{projectId}/survey/business-systems/{businessSystemId}/topology-diagram`
   - `POST /api/v1/projects/{projectId}/survey/business-systems/{businessSystemId}/data-flow-diagram`
   - 建议后端扩展文件白名单：图片类型 + `.vsdx`。
   - `.vsdx` 常见 MIME 为 `application/vnd.ms-visio.drawing.main+xml`，也可能由浏览器上传为 `application/octet-stream`，建议同时校验扩展名。

2. 数据处理活动整体问卷接口需要以 `involvedActivities` 为准。
   - `GET /api/v1/projects/{projectId}/survey/processing-activity-survey`
   - `PUT /api/v1/projects/{projectId}/survey/processing-activity-survey`
   - 当某个活动类型未出现在 `involvedActivities` 中时，后端应忽略或清空对应环节字段。

3. 安全防护措施接口建议按控制字段清理补充字段。
   - `PUT /api/v1/projects/{projectId}/survey/security-protection`
   - 对“是/否”控制字段，值不是 `YES` 时，对应补充字段建议保存为空字符串或 `null`。
   - 重点字段关系：
     - `classifiedProtectionDone` -> `classifiedProtectionLevel`
     - `securityTestingInRecent3Years` -> `securityTestingTimes`
     - `historicalIssuesRectified` -> `unrectifiedIssueDescription`
     - `dataSecurityOrgEstablished` -> `dataSecurityOrg`
     - `dataSecurityOwnerAppointed` -> `dataSecurityOwner`

4. 创建项目接口不需要自动进入评估流程。
   - `POST /api/v1/projects` 创建成功后仍只需返回 `projectId`。
   - 项目开始评估仍由 `POST /api/v1/projects/{projectId}/start` 触发。
   - 后端不应在创建项目时生成检查项快照，检查项快照仍应在开始评估时生成。
