# 汇总分析功能修改说明

更新时间：2026-06-02

## 1. 数据来源调整

- 数据安全风险源清单的数据来源由仅包含 `evaluationResult = NON_COMPLIANT` 调整为包含：
  - `NON_COMPLIANT`：不符合
  - `PARTIAL`：基本符合
- 刷新统计数据时，一条“基本符合”或“不符合”的现场测评记录对应一条 `project_risk_summary_record` 风险汇总记录。

## 2. 国标风险源模板

- 新增数据库表 `risk_source_template`，用于存储 `doc/国标风险源模版.xlsx` 内容。
- 模板匹配关键属性：
  - `sheet_name`
  - `评估类别`
  - `评估子类`
  - `评估项`
- 刷新统计数据时，后端使用项目检查项的 `sheetName`、`category`、`subcategory`、`checkPoint` 与模板匹配。
- 匹配成功后，将模板字段填充到 `project_risk_summary_record`：
  - 模板“问题描述” -> `riskDescription`
  - 模板“整改建议” -> `remediationSuggestion`
  - 模板“常见风险源” -> `riskSourceDescription`
  - 模板“风险类型” -> `riskTypes`

## 3. 默认值与发生可能性调整

- `relatedData`（涉及的数据及类型、级别）默认值为 `"-"`。
- `harmLevel`（危害程度）默认值为 `"-"`。
- `possibilityLevel`（发生可能性）不再固定为 `"-"`，刷新统计数据时按现场测评分数和项目绑定评分模型的 `possibilityRanges` 自动填充。
- `riskLevel`（风险等级）默认值为空。
- 分数计算口径与现场测评“分数计算”接口一致，`NOT_APPLICABLE` 不计入分子和分母。
- 发生可能性区间按 `min <= score < max` 判断；为覆盖满分，`score = 100` 且区间 `max = 100` 时命中该区间。

## 4. 修改功能删除项

### 4.1 数据安全风险清单页

修改弹窗/保存请求中删除以下字段：

- `riskTypes`：风险类型
- `riskDescription`：风险描述
- `riskSourceDescription`：风险源描述
- `relatedData`：涉及的数据及类型、级别
- `relatedActivities`：涉及的数据处理活动

保留可修改字段：

- `harmLevel`
- `possibilityLevel`
- `riskLevel`

### 4.2 数据安全风险处置建议页

修改弹窗/保存请求中删除以下字段：

- `riskDescription`：风险描述
- `riskSourceDescription`：风险源描述
- `riskLevel`：风险等级

保留可修改字段：

- `remediationSuggestion`

## 5. 接口兼容说明

- 三个列表接口仍保持不变，并支持分页参数 `pageNo`、`pageSize`，统一返回 `{ list, pageNo, pageSize, total }`：
  - `GET /api/v1/projects/{projectId}/risk-sources?pageNo=1&pageSize=10`
  - `GET /api/v1/projects/{projectId}/risk-items?pageNo=1&pageSize=10`
  - `GET /api/v1/projects/{projectId}/risk-suggestions?pageNo=1&pageSize=10`
- 三个修改接口仍保持不变，但后端会忽略已删除的字段：
  - `PUT /api/v1/projects/{projectId}/risk-sources/{riskSourceId}`
  - `PUT /api/v1/projects/{projectId}/risk-items/{riskItemId}`
  - `PUT /api/v1/projects/{projectId}/risk-suggestions/{suggestionId}`
- `riskSourceId`、`riskItemId`、`suggestionId` 仍统一映射为同一个 `riskRecordId`。
