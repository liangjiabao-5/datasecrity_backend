# 数据库概要设计

## 1. 设计原则

- 项目是主数据，所有评估过程数据必须关联 `project_id`。
- 知识库中的模板、模型、矩阵和建议必须版本化。
- 项目引用知识库配置时保存快照，避免后续知识库修改影响历史项目。
- 删除默认采用逻辑删除。
- 关键业务表保留创建人、创建时间、更新人、更新时间。
- 报告生成、评分计算、风险刷新等关键操作要保留历史记录。

`matrix_json` 行为风险发生可能性：`HIGH`、`MEDIUM`、`LOW`。列为风险危害程度：`VERY_HIGH`、`HIGH`、`RELATIVELY_HIGH`、`MEDIUM`、`LOW`。单元格值为风险等级：`MAJOR`、`HIGH`、`MEDIUM`、`LOW`、`SLIGHT`。

知识库版本字段建议：

| 字段 | 说明 |
| --- | --- |
| version | 版本号 |
| status | 启用、停用、草稿 |
| effective_at | 生效时间 |
| superseded_by | 被哪个版本替代 |

## 12. 索引建议

- `project.project_code` 普通索引，用于编号查询；允许重复。
- 所有项目子表建立 `project_id` 索引。
- 现场测评表建立 `project_id + item_id` 索引。
- 风险表建立 `project_id + risk_level` 索引。
- 报告表建立 `project_id + status` 索引。
- 知识库表建立 `status + version` 索引。
