# 研发任务拆解与实施计划

## 1. 交付目标

本计划用于指导研发落地。目标是在保证技术边界清晰的前提下，将需求说明和原型拆解为前端、后端、报告服务、大模型辅助、测试和项目管理任务。

一期目标是完成项目评估闭环：

```text
项目创建 -> 项目基本信息 -> 评估方案 -> 信息调研 -> 现场测评 -> 汇总分析 -> 报告生成 -> 报告下载
```

知识库贯穿全流程，优先完成模板、评分模型、风险危害程度模型、风险评价矩阵、处置建议和案例基础能力。

## 3. 任务分解

### 3.1 基础工程

前端：

- 初始化 Vue 3 + TypeScript + Vite 工程。
- 接入 Element Plus、Vue Router、Pinia、Axios。
- 搭建主布局、左侧导航、顶部栏、项目内页面头部。
- 封装请求、分页表格、工具栏、弹窗、文件上传、字典下拉。

后端：

- 初始化 Python Flask 工程，采用 app factory、Blueprint、SQLAlchemy、Alembic/Flask-Migrate 的工程结构。
- 统一响应、异常、分页、审计、权限框架。
- 数据库迁移脚本框架。
- 接入对象存储、Redis、Celery/RQ 异步任务。
- 输出 OpenAPI 文档。

联调：

- 完成登录态或临时用户上下文。
- 完成权限占位和接口联通。

### 3.2 项目管理

前端：

- 项目管理列表、状态筛选、统计卡片。
- 创建项目页面。
- 编辑项目页面。
- 项目详情弹窗、团队成员、存入模板。

后端：

- 项目主表、项目配置快照。
- 创建、编辑、详情、列表、统计、删除、开始评估。
- 项目模板保存。
- 项目状态流转。

联调重点：

- 项目编号可重复场景，以及所有业务操作以项目 ID 定位。
- 开始评估后生成检查项快照。
- 修改模型/模板时的影响提示。

### 3.3 项目基本信息

前端：

- 项目基本情况标签页。
- 被评估单位基本信息标签页。
- 法规/标准多选和自定义新增。
- 导入、导出、保存、下一步。

后端：

- 项目基本信息保存和查询。
- 法规、标准引用。
- 项目自定义法规/标准。
- 联系人列表。

联调重点：

- 自定义条目是否写入项目私有数据。
- 保存后报告生成能读取到最新数据。

### 3.4 评估方案

前端：

- 评估团队、被评估方团队、关注点、差距项四个标签页。
- 新增、修改、删除弹窗。
- 查询、重置、导入、导出。
- 选择已有案例。

后端：

- 团队成员 CRUD。
- 被评估方团队 CRUD。
- 关注点 CRUD。
- 差距项 CRUD。
- 案例选择接口。

联调重点：

- 四类数据的分页、导入和导出。
- 删除时权限和引用校验。

### 3.5 知识库管理

前端：

- 评估项模板设置。
- 系统模板、数据处理活动、安全防护措施。
- 风险处置建议管理。
- 评分模型管理。
- 风险危害程度分析模型管理。
- 风险评价矩阵管理。
- 参考案例维护。

后端：

- 知识库条目 CRUD。
- 模板、模型、矩阵版本管理。
- 评分模型区间校验。
- 风险评价矩阵校验。
- 被引用条目停用逻辑。

联调重点：

- 评分模型区间覆盖 0%-100%，无重叠、无空档。
- 项目引用知识库后保存版本快照。

### 3.6 信息调研

前端：

- 业务和信息系统调研。
- 数据资产表格。
- 个人信息、重要数据、核心数据。
- 七类数据处理活动。
- 安全防护措施调研。
- 拓扑图、数据流转图上传。

后端：

- 业务系统 CRUD。
- 数据资产 CRUD。
- 个人信息/重要数据/核心数据 CRUD。
- 数据处理活动 CRUD。
- 安全防护措施保存。
- 文件上传关联。

联调重点：

- 数据资产和业务系统关联。
- 七类处理活动统一模型。
- 导入错误明细。

### 3.7 现场测评

前端：

- 检查项目录。
- 检查项列表。
- 评估结果筛选。
- 批量修改评估结果。
- 检查记录编辑弹窗。
- 分数计算结果弹窗。
- 参考案例对比分析和应用建议。

后端：

- 项目检查项快照。
- 检查项查询。
- 评估记录保存。
- 批量修改。
- 分数计算。
- 导出模板、导入、导出。

联调重点：

- 不适用项不参与评分。
- 批量修改审计。
- 分数计算使用项目评分模型版本。

### 3.8 汇总分析

前端：

- 风险源清单。
- 数据安全风险清单。
- 风险处置建议。
- 修改弹窗。
- 刷新统计数据。

后端：

- 从基本符合、不符合现场测评记录生成一对一风险汇总记录。
- 通过国标风险源模板匹配填充评估类别、评估子类、风险描述、整改建议、风险源描述、风险源类型和风险类型。
- 风险源清单、风险清单和处置建议从同一条风险汇总记录投影生成。
- 风险等级计算。
- 处置建议生成或匹配。
- 人工修改保存。

联调重点：

- 刷新统计数据不覆盖人工确认数据。
- 风险可追溯来源检查项和数据资产。
- 风险等级自动计算和人工修改并存。

### 3.9 报告管理

前端：

- 报告列表。
- 报告生成配置。
- 生成状态轮询。
- 下载、删除、分页。

后端：

- 报告任务。
- 报告记录。
- Word 模板填充。
- 表格/图示编号。
- 文件存储和下载。

联调重点：

- 生成失败提示。
- 下载权限。
- 多次生成多份报告记录。

### 3.10 大模型辅助

前端：

- 现场测评案例建议区域。
- 应用建议按钮。
- 整改建议生成入口。
- 报告段落草稿入口，可作为后续增强。

后端：

- 大模型网关。
- 脱敏、审计。
- 案例匹配。
- 评估记录建议。
- 整改建议生成。

联调重点：

- 大模型不可用不影响主流程。
- 输出必须人工确认后入库。
- 审计记录完整。

## 4. 前后端责任边界

| 能力 | 前端责任 | 后端责任 |
| --- | --- | --- |
| 表单校验 | 必填、格式、长度、区间前置校验 | 最终业务校验、唯一性、权限、状态 |
| 分页查询 | 传递页码、页大小、筛选条件 | 返回分页数据和总数 |
| 状态筛选 | 展示标签和切换条件 | 按状态查询并返回统计 |
| 导入 | 上传文件、展示错误明细 | 文件解析、数据校验、错误报告 |
| 导出 | 触发导出、下载文件 | 生成文件、权限校验 |
| 分数计算 | 触发计算、展示结果 | 按模型计算并保存记录 |
| 风险等级 | 展示自动/人工标识 | 按矩阵计算、保存调整记录 |
| 报告生成 | 选择章节、轮询状态 | 异步生成、存储文件、返回状态 |
| 大模型建议 | 展示建议、应用建议 | 脱敏、调用模型、校验输出、审计 |

## 5. 里程碑计划

### 里程碑 1：工程与基础能力

周期建议：第 1-2 周。

交付：

- 前后端基础工程。
- 统一接口规范。
- 登录和权限占位。
- 项目列表、创建、编辑基础接口和页面。

验收：

- 能创建项目并进入项目流程。
- 接口响应、错误和分页格式统一。

### 里程碑 2：项目配置与知识库基础

周期建议：第 3-4 周。

交付：

- 项目基本信息。
- 评估方案。
- 知识库模板、评分模型、危害模型、矩阵基础维护。

验收：

- 项目可绑定模板、模型和矩阵。
- 评分模型区间校验有效。

### 里程碑 3：调研与现场测评

周期建议：第 5-7 周。

交付：

- 信息调研四个标签页。
- 现场测评检查项、评估记录、批量修改、分数计算。
- 导入导出基础能力。

验收：

- 可完成一个项目的调研和测评录入。
- 分数计算结果正确。

### 里程碑 4：汇总分析与报告生成

周期建议：第 8-10 周。

交付：

- 风险源清单、风险清单、处置建议。
- 风险等级计算。
- 报告任务、报告生成、下载。

验收：

- 可从项目数据生成报告。
- 报告管理显示生成状态。

### 里程碑 5：增强与验收

周期建议：第 11-12 周。

交付：

- 大模型辅助。
- 审计日志。
- 权限细化。
- 性能优化和回归测试。

验收：

- 大模型不可用时主流程正常。
- 关键操作有审计记录。
- 用户验收问题闭环。

## 6. 测试策略

### 6.1 单元测试

重点覆盖：

- 评分模型区间校验。
- 分数计算。
- 风险评价矩阵计算。
- 项目状态流转。
- 报告任务状态。

### 6.2 接口测试

重点覆盖：

- 项目 CRUD。
- 信息调研 CRUD。
- 现场测评保存、批量修改。
- 汇总分析刷新和人工修改。
- 报告生成、下载、删除。
- 知识库条目被引用后的停用逻辑。

### 6.3 前端测试

重点覆盖：

- 标签页切换。
- 表单校验。
- 弹窗保存和取消。
- 导入错误展示。
- 报告状态轮询。

### 6.4 端到端测试

核心用例：

1. 创建项目。
2. 完善项目基本信息。
3. 维护评估团队。
4. 录入信息调研。
5. 完成现场测评。
6. 计算分数。
7. 生成风险清单。
8. 生成并下载报告。

## 7. 主要风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 报告模板复杂度高 | 排期延误 | 先实现核心章节，复杂格式分阶段 |
| 导入模板不稳定 | 数据校验困难 | 固定模板版本，导入返回错误明细 |
| 知识库频繁变化 | 项目结果不可复现 | 项目引用时保存版本快照 |
| 风险自动生成不符合预期 | 用户信任度下降 | 自动生成草稿，人工确认后入报告 |
| 大模型输出不稳定 | 影响建议质量 | 可关闭、可重试、人工确认、保留规则建议 |
| 页面表单过多 | 前端维护成本高 | 统一表单、表格、弹窗组件 |

- OpenAPI 接口文档。
- 报告模板。
- 导入模板。
- 部署文档。
- 测试报告。
- 用户操作说明。
- 初始化知识库数据。

## 9. 启动前待确认事项

以下事项建议在研发启动会确认：

- 最终采用 MySQL 还是 PostgreSQL。
- 报告模板是否已有正式版。
- 评估项模板和国标测评录入模板是否作为初始化数据导入。
- 是否一期接入大模型，接入外部 API 还是私有化模型。
- 权限体系是否接入现有用户系统。
- 是否需要历史项目数据迁移。
