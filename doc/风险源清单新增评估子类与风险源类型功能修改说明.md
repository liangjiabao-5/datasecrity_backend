# 风险源清单新增评估子类与风险源类型功能修改说明

## 1. 背景

“数据安全风险源清单”需要根据 `doc/国标风险源模版.xlsx` 展示评估子类、风险源类型。后端已在项目风险汇总记录中保存评估子类，并从风险源模板同步风险源类型，在风险源清单接口和导出文件中返回。

## 2. 后端字段

| 中文列名 | 接口字段 | 来源 | 说明 |
| --- | --- | --- | --- |
| 评估子类 | `assessmentSubcategory` | 国标风险源模板“评估子类” | 只读展示 |
| 风险源类型 | `riskSourceType` | 国标风险源模板“风险源类型” | 只读展示 |
| 风险类型 | `riskTypes` | 国标风险源模板“风险类型” | 原有字段，位置调整 |

## 3. 受影响接口

### 风险源清单

```http
GET /api/v1/projects/{projectId}/risk-sources?pageNo=1&pageSize=10
```

`data.list[]` 新增：

```json
{
  "assessmentSubcategory": "数据安全制度体系",
  "riskSourceType": "数据安全制度流程存在缺陷"
}
```

### 风险源导出

```http
GET /api/v1/projects/{projectId}/risk-sources/export
```

导出列顺序调整为：

| 顺序 | 列名 | 字段 |
| --- | --- | --- |
| 1 | 序号 | index |
| 2 | 评估项ID | assessmentItemId |
| 3 | 评估子类 | assessmentSubcategory |
| 4 | 检查要点 | checkPoint |
| 5 | 评估结果 | evaluationRecord |
| 6 | 风险描述 | riskDescription |
| 7 | 风险源描述 | riskSourceDescription |
| 8 | 风险源类型 | riskSourceType |
| 9 | 风险类型 | riskTypes |
| 10 | 涉及的数据及类型、级别 | relatedData |
| 11 | 涉及的数据处理活动 | relatedActivities |
| 12 | riskSourceId | 隐藏定位列 |

### 风险源导入

```http
POST /api/v1/projects/{projectId}/risk-sources/import
```

导入兼容新增列。“评估子类”为只读校验列，用户修改后会返回行级错误；“风险源类型”为模板展示列，导入时不作为可编辑字段处理。

## 4. 前端调整要求

- 风险源清单表格展示 `assessmentSubcategory` 列，放在 `assessmentItemId` 后面。
- 将 `riskSourceType` 和原有 `riskTypes` 列移动到 `riskSourceDescription` 后面。
- `riskTypes` 是数组，继续按当前方式拼接展示。
- `assessmentSubcategory`、`riskSourceType` 作为只读展示字段，不需要在保存风险源时提交。

## 5. 联调验证

1. 刷新汇总分析后，风险源清单每行返回 `assessmentItemId`、`assessmentSubcategory`、`riskSourceType`、`riskTypes`。
2. 风险源清单列顺序符合：评估项ID、评估子类、检查要点、评估结果、风险描述、风险源描述、风险源类型、风险类型。
3. 导出风险源 Excel 后，前 11 列顺序与接口说明一致，`riskSourceId` 仍为隐藏列。
4. 修改风险源保存时，不提交 `assessmentSubcategory` 和 `riskSourceType` 也能正常保存原有可编辑字段。
