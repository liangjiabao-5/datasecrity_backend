# 流程页操作区功能修改说明

更新时间：2026-05-28

## 修改范围

本次前端调整涉及以下项目流程页面：

- 项目基本信息
- 评估方案
- 信息调研
- 现场测评
- 汇总分析
- 报告管理

## 删除的前端功能/结构

1. 删除流程页内容区顶部的重复标题操作条。
   - 删除内容包括页面内重复展示的页面标题、项目名称、导入、导出、保存、下一步等按钮。
   - 页面标题和当前项目信息保留在全局顶部栏展示。

2. `ProjectHeader` 组件不再承载页面操作按钮。
   - `ProjectHeader` 现在只负责展示项目流程步骤导航。
   - 各页面按钮改为放在各自内容面板的操作区。

## 修改的前端功能

### 1. 项目基本信息

按钮调整到项目基本信息面板顶部：

- 导入：导入当前页面 JSON 数据，导入后需点击保存生效。
- 导出：导出当前页面表单 JSON 数据。
- 保存：调用 `PUT /api/v1/projects/{projectId}/basic-info`。
- 下一步：保存成功后跳转评估方案。

后端建议：

- 如需正式文件导入导出，请补充项目基本信息导入/导出接口。
- 当前前端 JSON 导入导出是无后端接口时的本地兜底能力。

### 2. 评估方案

按钮调整到评估方案面板顶部：

- 导入：导入评估团队、被评估方团队、关注点、差距项 JSON 数据。
- 导出：导出以上四类当前数据。
- 保存：刷新当前四类数据并提示已保存；各记录的新增、修改、删除仍实时调用对应 CRUD 接口。
- 下一步：跳转信息调研。

后端建议：

- 如需正式导入导出，请补充评估方案整体导入/导出接口，或为四类子资源分别提供导入/导出接口。

### 3. 信息调研

按钮调整到信息调研面板顶部：

- 导入：导入当前标签页 JSON 数据。
- 导出：导出当前标签页 JSON 数据。
- 保存：按当前标签页保存业务和信息系统、数据资产、数据处理活动问卷或安全防护措施问卷。
- 下一步：跳转现场测评。

后端建议：

- 继续参考 `doc/信息调研功能修改说明.md`，补齐数据处理活动整体问卷接口和安全防护措施结构化字段。
- 如需正式导入导出，请按当前标签页粒度提供接口或任务化导入导出能力。

### 4. 现场测评

按钮调整到现场测评页面操作区：

- 导出模板：打开 `GET /api/v1/projects/{projectId}/evaluation/export-template`。
- 导入：上传文件到 `POST /api/v1/projects/{projectId}/evaluation/import`。
- 导出：打开 `GET /api/v1/projects/{projectId}/evaluation/export`。
- 分数计算：调用 `POST /api/v1/projects/{projectId}/evaluation/calculate-score`。
- 下一步：跳转汇总分析。

后端建议：

- 当前文档中现场测评导入导出为占位实现，需补充真实文件生成、文件解析、错误明细返回和下载能力。

### 5. 汇总分析

按钮调整到汇总分析面板顶部：

- 刷新统计数据：调用 `POST /api/v1/projects/{projectId}/risk-summary/refresh`。
- 下一步：跳转报告管理。

后端建议：

- 保持刷新统计接口支持 `overwriteManualChanges`，前端已按用户确认结果传递该参数。

### 6. 报告管理

按钮调整到报告管理面板顶部：

- 生成：调用 `POST /api/v1/projects/{projectId}/reports/generate`。
- 刷新：调用 `GET /api/v1/projects/{projectId}/reports`。

后端实现：

- 已补充生成前就绪检查、真实后台任务创建、任务状态查询、失败重试、报告列表、Word 下载和逻辑删除能力。
- 报告生成读取项目最终确认数据，包括项目基本信息、评估方案、信息调研、现场测评、汇总分析结果。
- 前端点击生成后应按任务接口轮询，具体流程见 `doc/报告管理功能开发说明.md`。

## 统一后端建议

1. 导入/导出类接口建议返回任务 ID，并提供任务状态与错误明细下载接口。
2. 导入操作应按页面或标签页校验字段，并返回行号、字段名、错误原因。
3. 导出操作应按当前项目权限校验，文件建议进入对象存储并返回短时下载链接。
4. 保存、刷新、生成、删除等操作应记录审计日志，便于报告追溯。
