# 风险清单按现场测评顺序排序功能修改说明

## 一、修改范围

以下三个标签页的列表顺序统一调整为现场测评页面的表格顺序：

- 数据安全风险源清单
- 数据安全风险清单
- 数据安全风险处置建议

对应接口保持不变：

```text
GET /api/v1/projects/{projectId}/risk-sources
GET /api/v1/projects/{projectId}/risk-items
GET /api/v1/projects/{projectId}/risk-suggestions
```

## 二、后端排序规则

后端通过风险记录关联的现场测评项进行排序：

1. 优先按现场测评项 `sortOrder` 升序排列。
2. `sortOrder` 相同时，按现场测评项 ID 升序排列，保证分页查询顺序稳定。
3. 开启“是否合并”后，数据安全风险清单和数据安全风险处置建议仍按现场测评顺序聚合；合并行在其第一条现场测评数据出现的位置展示，合并行内的评估项 ID、风险描述和评估结果也按现场测评顺序拼接。

## 三、前端调整建议

本次修改不新增接口、不新增字段、不改变三个列表的表结构。

前端应直接按照接口响应中 `list` 数组的顺序渲染，不要再按风险记录的创建时间、更新时间、评估项 ID 或其他字段进行二次排序。分页参数和现有合并行处理逻辑保持不变。

## 四、验收建议

1. 在现场测评页面选择多条顺序不连续的数据，设置为“基本符合”或“不符合”。
2. 刷新汇总分析后，确认三个标签页中的数据顺序与这些数据在现场测评页面中的先后顺序一致。
3. 开启“是否合并”，确认数据安全风险清单和数据安全风险处置建议中的合并行顺序仍与现场测评顺序一致。
4. 分页查看列表，确认翻页前后无重复、遗漏或顺序跳变。
