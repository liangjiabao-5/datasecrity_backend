# 项目基本信息与信息调研原型变更功能修改说明

更新时间：2026-06-16

本文用于说明前端已根据 `原型/项目基本信息.html` 和 `原型/信息调研.html` 完成的字段变更，供后端同步接口、数据库字段、导入导出模板和 DTO。

## 1. 项目基本信息

接口仍使用：

- `GET /projects/{projectId}/basic-info`
- `PUT /projects/{projectId}/basic-info`
- `GET /projects/{projectId}/basic-info/export-template`
- `GET /projects/{projectId}/basic-info/export`
- `POST /projects/{projectId}/basic-info/import`

### 1.1 新版 payload

```json
{
  "projectNumber": "ENST-2024-001",
  "projectName": "XX银行数据安全评估项目",
  "laws": [{ "id": "law-001", "name": "《中华人民共和国数据安全法》" }],
  "standards": [{ "id": "std-001", "name": "GB/T 45577-2025《数据安全技术 数据安全风险评估方法》" }],
  "assessmentPlan": {
    "startDate": "2026-06-01",
    "endDate": "2026-06-30"
  },
  "organization": {
    "name": "XX银行股份有限公司",
    "postalCode": "100000"
  },
  "contacts": [
    {
      "id": "contact-001",

      "phone": "010-88888888",
      "email": "zhangsan@example.com"

```

### 1.2 字段变更

新增或保留：

- `projectNumber`：项目编号，前端只读展示；如后端已有 `projectCode`，可在返回时映射为 `projectNumber`。
- `projectName`：项目名称。
- `laws`、`standards`、`assessmentPlan.startDate`、`assessmentPlan.endDate` 保留。
- `organization.name`、`organization.postalCode` 保留。
- `contacts` 保留，字段不变。

删除：

- `projectDescription`
- `systemDescription`
- `assessmentTarget`
- `organization.creditCode`
- `organization.address`
- `organization.dataSecurityOwner`
- `organization.description`

后端导入导出模板需要同步为两个页签：`项目基本情况`、`被评估单位基本信息/联系人信息`。由于 Excel 工作表名称不允许包含 ASCII `/`，实际 xlsx 页签使用 `被评估单位基本信息／联系人信息`。

数据库同步要求：

- `project_basic_info` 删除 `project_description`、`system_description`、`assessment_target`。
- `assessed_organization` 删除 `address`、`credit_code`、`data_security_owner`、`description`。
- 新版项目名称直接保存到 `project.project_name`，项目编号继续读取 `project.project_code`。

## 2. 信息调研

### 2.1 新增：数据处理者基本情况调研

新增接口建议：

- `GET /projects/{projectId}/survey/data-processor-basic`
- `PUT /projects/{projectId}/survey/data-processor-basic`

payload：

```json
{
  "unitName": "",
  "unifiedSocialCreditCode": "",
  "officeAddress": "",
  "legalRepresentative": "",
  "staffSize": "",
  "businessScope": "",
  "dataSecurityOfficer": "",
  "contactInfo": "",
  "unitNature": "",
  "specificProcessorType": "",
  "powerIndustryCategory": "",
  "businessOperationArea": "",
  "dataProcessingLocation": "",
  "mainBusinessScope": "",
  "businessScale": "",
  "administrativeLicense": ""
}
```

`unitNature` 可选值：`党政机关`、`事业单位`、`企业`、`社会团体`、`其他`。

数据库表 `data_processor_basic_survey` 必须按以上字段逐列存储，不使用整页 `payload` JSON。

### 2.2 业务和信息系统调研

既有接口保持：

- `GET /projects/{projectId}/survey/business-systems`
- `POST /projects/{projectId}/survey/business-systems`
- `PUT /projects/{projectId}/survey/business-systems/{id}`

`BusinessSystem` 新增字段：

```json
{
  "dataScopes": ["一般数据", "个人信息", "重要数据", "核心数据"]
}
```

数据库列 `business_system.data_scopes` 使用文本列保存多选结果，接口层转换为数组返回。

其他字段继续保留：`systemName`、`businessFunction`、`serviceObject`、`userScale`、`coverageArea`、`classifiedProtectionLevel`、`relatedDepartments`、`topologyFileId`、`businessFlowFileId`。

拓扑图和数据流转图上传接口不变，前端仍按 `.vsdx` 文件上传。

数据库表 `security_protection_survey` 必须按以上字段逐列存储，不再使用整页 `payload` JSON。

## 3. JSON 导入导出

信息调研前端 JSON 导入导出新增 `processorBasic`：

```json
{
  "processorBasic": {},
  "business": {},
  "dataAssets": [],
  "personalInfo": [],
  "importantData": [],
  "coreData": [],
  "processing": {},
  "security": {}
}
```

后端如提供服务端导入导出，需要同步这些 key。

## 4. 兼容建议

- GET 接口返回缺失字段时，前端会以空字符串或空数组兜底。
- PUT 接口请允许新版 payload 不包含已删除字段。
- 数据库不再保留本次删除字段；如历史库存在旧字段，运行时结构同步会在新增列和回填后删除。
- 报告管理默认报告名已由旧 `assessmentTarget` 改为读取基本信息中的 `projectName`。
