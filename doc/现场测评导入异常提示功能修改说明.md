# 现场测评导入异常提示功能修改说明

更新时间：2026-06-10

## 修改背景

现场测评页面调用 `POST /api/v1/projects/{projectId}/evaluation/import` 导入 Excel 时，原逻辑对部分异常只返回泛化错误，且当导入模板中的定位列被误编辑时，仍可能覆盖原有现场测评记录。

本次调整后，导入接口会在写入前做行级校验，并在返回结果中明确指出异常所在行、字段和原因。

## 删除功能

无。

## 修改功能

### 1. 符合情况导入值校验

`符合情况` 列仅支持以下中文值或系统枚举值：

- `符合` / `COMPLIANT`
- `基本符合` / `PARTIAL`
- `不符合` / `NON_COMPLIANT`
- `不适用` / `NOT_APPLICABLE`

当导入值不在上述范围内时，该行不写入数据库，并返回错误明细，例如：

```json
{
  "rowNo": 2,
  "field": "符合情况",
  "reason": "符合情况列只能填写符合、基本符合、不符合、不适用或对应枚举值 COMPLIANT、PARTIAL、NON_COMPLIANT、NOT_APPLICABLE。"
}
```

### 2. 模板定位列误编辑校验

导入时会校验以下模板定位列是否与系统中当前项目的检查项快照一致：

- 检查项ID
- 检查项编号
- 工作表
- 一级分类
- 二级分类
- 检查要点

如果这些列被误编辑，该行不写入数据库，不会覆盖原有现场测评记录，并返回具体异常位置，例如：

```json
{
  "rowNo": 2,
  "field": "检查项编号",
  "reason": "检查项编号与系统检查项不一致，请使用最新导出模板。"
}
```

同一行中多个定位列异常时，会返回多条错误明细。

### 3. 保留行级部分导入模式

接口仍沿用原有同步行级导入方式：

- 正常行继续导入。
- 异常行跳过，不覆盖原数据。
- `failedCount` 表示异常明细数量。
- `errors` 数组返回异常位置和原因。

返回结构示例：

```json
{
  "projectId": "p-001",
  "importedCount": 0,
  "failedCount": 1,
  "errors": [
    {
      "rowNo": 2,
      "field": "符合情况",
      "reason": "符合情况列只能填写符合、基本符合、不符合、不适用或对应枚举值 COMPLIANT、PARTIAL、NON_COMPLIANT、NOT_APPLICABLE。"
    }
  ]
}
```

## 前端适配建议

需要前端确认或补充展示导入错误明细：

- 上传后如果 `data.failedCount > 0`，应展示 `data.errors`。
- 建议展示字段：`rowNo`、`field`、`reason`。
- 可提示用户“请下载最新模板或检查提示行后重新导入”。
- 接口地址、请求方式、文件字段名没有变化。
- 返回结构保持原有 `importedCount`、`failedCount`、`errors`，但 `field` 现在返回中文列名，`reason` 返回可直接展示的中文说明。
